# HTML Canvas 动画项目开发规范

> 从「离子反应」项目中总结的通用性错误和规范，适用于所有化学教学动画项目。

---

## 核心原则

### 优先级

1. **功能准确性** — 化学原理正确、动画逻辑无误、交互行为符合预期
2. **视觉准确美观** — 配色协调、布局合理、动画流畅自然

### 文件组织

- **多文件优先**：当项目复杂度需要时，拆分为 HTML + JS + CSS 多文件，各司其职
- **单文件为优化**：只有当单文件能达到与多文件完全相同的效果时，才合并为单文件
- **不因形式牺牲质量**：不为了"单文件好分发"而把代码塞成一坨不可维护的长文本

### 依赖使用

- **优先原生实现**：HTML + CSS + Canvas + 原生 JS 已能覆盖绝大多数教学动画需求，优先用原生方案
- **允许外部依赖**：当实现复杂效果（如 3D 渲染、物理引擎、高级图表）原生代码过于复杂或不可行时，可引入外部库
- **最小化原则**：只引入必要的库，用 CDN 链接而非 npm 构建，确保双击 HTML 仍可直接运行

### 字体规范

- **所有页面字体不小于 28px**（按钮、信息栏、标题等），确保课堂投影清晰可读

### 画布尺寸

- **目标设备分辨率 1920×1080**，HTML 页面应匹配此尺寸
- 页面总高度（含按钮、信息栏等）不能超出 1080px，避免出现滚动条
- 页面总宽度不能超出 1920px，确保全屏投影无裁切

### 文档存放

- **所有文档（计划、方案、规范等）统一放在 `docs/` 目录下**，不要在工程文件夹（如 `必修一/`、`必修二/`）内生成 .md 文件

### 计划与验收

- **每个计划文档末尾必须附验收清单**，列出所有可验证的检查项
- **代码实现完成后，逐项验收通过才算完成**，未通过项需修复后重新验收

### 可分发性

课堂场景需要双击即用，但这不等于必须单文件：

- 多文件项目打包为文件夹，双击 HTML 即可打开
- 无需服务器、无需 npm、无需构建工具
- 路径全部用相对路径，移动文件夹不破坏引用
- 若使用外部依赖，优先 CDN 引入，避免构建步骤

## 一、通用性错误清单

### 1. const 引用时序问题

`const` 不会变量提升（unlike `var`），必须先定义再引用。

```javascript
// ❌ 错误：STEP_DURATIONS 引用了还没定义的 STEP0_HOLD
const STEP_DURATIONS = [STEP0_HOLD + STEP0_DISSOLVE, ...];
const STEP0_HOLD = 1.5;  // 定义在后面，报错

// ✅ 正确：先定义所有常量，再引用
const STEP0_HOLD = 1.5;
const STEP0_DISSOLVE = 2.0;
const STEP_DURATIONS = [STEP0_HOLD + STEP0_DISSOLVE, ...];
```

**教训：** 遇到 `ReferenceError`，第一反应检查 const/let 声明顺序。

---

### 2. 状态管理：累积 vs 独立

分步动画的状态管理有两种模式，选错会导致逻辑崩溃：

| 模式 | 适用场景 | 实现 |
|------|----------|------|
| **独立** | 每步互不影响 | 跳转时清空重建 |
| **累积** | 后一步依赖前一步的结果 | 跳转时回放历史状态 |

```javascript
// ❌ 错误：步骤2需要步骤1的离子，但跳转时清空了
function goToStep(step) {
  ions = [];  // 步骤1的离子全没了
  if (step === 2) initStep2();  // 找不到 Ba²⁺ 和 SO₄²⁻
}

// ✅ 正确：累积重建前面步骤的状态
function goToStep(step) {
  ions = [];
  if (step >= 1) addDissolvedIons(...);  // 回放步骤1的产物
  if (step >= 2) addDissolvedIons(...);  // 回放步骤2的产物
  if (step === 2) initStep2();            // 进入步骤2
}
```

**教训：** 设计分步动画时，先明确每步的输入依赖，再决定状态管理策略。

---

### 3. 对象匹配：索引 vs 距离

在动画中匹配对象（如配对离子），距离判断容易出错：

```javascript
// ❌ 错误：两个对象合并到同一位置后，距离为 0，无法区分
const partner = list.find(i => Math.abs(i.x - target.x) < 100);

// ✅ 正确：按索引配对，顺序确定性
const pairA = listA[i];
const pairB = listB[i];
```

**教训：** 能用索引/ID 确定关系时，不要用空间距离判断。

---

### 4. 运动状态机不完整

对象的状态转换必须覆盖所有路径，否则会出现"卡住"或"幽灵运动"：

```javascript
// ❌ 错误：sinking 结束后没有新状态，回到布朗运动
if (this.sinking) {
  this.y += 60 * dt;
  if (到达底部) this.sinking = false;
  // ← 回到默认分支，离子继续布朗运动
}

// ✅ 正确：添加 settled 标志，彻底停止
if (this.settled) return;
if (this.sinking) {
  this.y += 60 * dt;
  if (到达底部) { this.sinking = false; this.settled = true; }
  return;
}
```

**教训：** 每个运动状态结束时，必须明确"接下来去哪"。

---

### 5. 居中基准不一致

多个元素声称"居中"，但使用了不同的参考坐标系：

```javascript
// ❌ 错误：大标题用画布居中，步骤文字用容器居中
ctx.fillText('离子反应', W / 2, 50);              // 画布中心 960
ctx.fillText(step.name, CONTAINER.x + CONTAINER.w / 2, 60);  // 容器中心 760

// ✅ 正确：统一使用同一个居中基准
const cx = CONTAINER.x + CONTAINER.w / 2;  // 全局居中参考点
ctx.fillText('离子反应', cx, 50);
ctx.fillText(step.name, cx, 60);
```

**教训：** 布局时先定义全局参考点（如 `cx`, `cy`），所有元素基于同一参考。

---

### 6. 固定尺寸容器不适应内容

UI 元素使用固定像素大小，文字超出时溢出裁切：

```javascript
// ❌ 错误：固定 600px 宽的色块，长文字溢出
ctx.fillRect(cx - 300, ty - 30, 600, 80);

// ✅ 正确：根据内容宽度计算容器大小，或给足余量
ctx.fillRect(cx - 420, ty - 36, 840, 90);
```

**教训：** 色块/容器宽度应基于最大可能内容宽度计算，而非硬编码。

---

### 7. 进度条跳转的三个陷阱

进度条点击跳转涉及动画状态、进度条视觉、交互逻辑三者的配合，容易出错：

**陷阱一：`advanceStep` 不同步进度条**

`advanceStep` 保留已有离子（连贯过渡），但忘了更新 `totalElapsed` 和 `progressDisplay`，导致点击相邻下一步时进度条不动。

```javascript
// ❌ 错误：进度条没跟上
function advanceStep(step) {
  currentStep = step;
  stepPhase = 'ready';
  stepTimer = 0;
  if (step === 1) initStep1();
}

// ✅ 正确：同步进度条时间
function advanceStep(step) {
  currentStep = step;
  stepPhase = 'ready';
  stepTimer = 0;
  let t = 0;
  for (let i = 0; i < step; i++) t += STEP_DURATIONS[i];
  totalElapsed = t;
  progressDisplay = t;
  if (step === 1) initStep1();
}
```

**陷阱二：跳转后进度条渐进追赶而非瞬移**

`update()` 中的 lerp 平滑会让 `progressDisplay` 渐进追赶 `totalElapsed`。正常播放时这是优点（丝滑），但跳转后进度条会缓慢滑动而非跳到目标位置。

```javascript
// ❌ 错误：跳转后进度条慢慢滑过去
progressDisplay += (totalElapsed - progressDisplay) * Math.min(6 * dt, 1);

// ✅ 正确：差值过小时直接同步
const diff = totalElapsed - progressDisplay;
if (Math.abs(diff) < 0.05) progressDisplay = totalElapsed;
else progressDisplay += diff * Math.min(6 * dt, 1);
```

**陷阱三：点击当前步骤重置动画**

点击进度条当前步骤范围时，`goToStep(currentStep)` 会重置动画到该步骤起点，用户感知为"进度条跳回去了"。

```javascript
// ❌ 错误：点击当前步骤也触发跳转
if (ratio <= cumDur / TOTAL_DURATION) {
  goToStep(i);  // 重置当前步骤
}

// ✅ 正确：点击当前步骤无操作
if (ratio <= cumDur / TOTAL_DURATION) {
  if (i === currentStep) return;
  if (i === currentStep + 1) advanceStep(i);
  else goToStep(i);
}
```

**教训：** 进度条交互必须同时处理三个问题——状态同步、视觉即时性、边界守卫。

---

### 8. 浏览器兼容性

使用新 API 前确认目标浏览器支持：

```javascript
// ❌ 错误：roundRect 是 2023 年新 API，旧浏览器报错
ctx.roundRect(x, y, w, h, r);

// ✅ 正确：使用兼容写法
ctx.fillRect(x, y, w, h);  // 简单矩形
// 或用 arcTo 手动画圆角矩形
```

**教训：** Canvas 新 API（roundRect, Path2D 等）需检查兼容性，或提供降级方案。

## 二、浅色风格规范

### 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| 页面背景 | `#f0f2f5` | body |
| 卡片背景 | `#fff` | 包裹容器 |
| 容器填充 | `#f5f8ff` | 画布内容器 |
| 容器边框 | `#c0c8d4` | 外边框 |
| 文字主色 | `#333` | 标题 |
| 文字次色 | `#556` | 描述 |
| 文字辅助 | `#888` | 标签 |

### 离子配色（深色在浅底上更醒目）

| 离子 | 色值 | 发光色 |
|------|------|--------|
| Ba²⁺ | `#2e7d32` | `rgba(46,125,50,0.25)` |
| Na⁺ | `#7b1fa2` | `rgba(123,31,162,0.25)` |
| Cl⁻ | `#f9a825` | `rgba(249,168,37,0.25)` |
| SO₄²⁻ | `#c62828` | `rgba(198,40,40,0.25)` |
| BaSO₄ | `#fafafa` | `rgba(0,0,0,0.08)` |

### 原则

- 背景用白/浅灰，不用深色（课堂投影深色背景反而不清晰）
- 离子用深色高饱和，在浅底上形成强对比
- 阴影用 `rgba(0,0,0,0.08)` 级别，轻柔不脏

## 三、视频播放器式进度条实现

### 设计目标

进度条像视频播放器一样：一条连续轨道，蓝色填充匀速推进，圆形滑块跟随，步骤标签均匀分布。

### 数据模型

```javascript
// 每步时长
const STEP_DURATIONS = [3.5, 3.5, 5.0]; // 秒
const TOTAL_DURATION = 12.0;

// 时间追踪
let totalElapsed = 0;    // 真实流逝时间（恒速推进）
let progressDisplay = 0; // 显示时间（lerp 平滑追赶）
```

### 核心逻辑

**推进（每帧更新）：**
```javascript
function update(dt) {
  totalElapsed += dt;  // 匀速
  const diff = totalElapsed - progressDisplay;
  if (Math.abs(diff) < 0.05) progressDisplay = totalElapsed;  // 差值过小时直接同步
  else progressDisplay += diff * Math.min(6 * dt, 1);  // 否则 lerp 平滑追赶
}
```

**绘制：**
```javascript
function drawProgressBar() {
  const ratio = progressDisplay / TOTAL_DURATION;
  // 轨道底色
  ctx.fillRect(x, barY, w, h);
  // 步骤分隔线
  drawSeparators();
  // 蓝色填充
  ctx.fillRect(x, barY, w * ratio, h);
  // 滑块圆点
  ctx.arc(x + w * ratio, barY + h/2, dotR, ...);
  // 步骤标签（按时间均匀分布）
  drawStepLabels();
}
```

**跳转时时间定位：**
```javascript
function goToStep(step) {
  let t = 0;
  for (let i = 0; i < step; i++) t += STEP_DURATIONS[i];
  totalElapsed = t;       // 直接跳到目标时间
  progressDisplay = t;    // 显示也立即到位
}
```

### 连贯过渡规则

| 操作 | 过渡方式 | 说明 |
|------|----------|------|
| 自动完成进入下一步 | `advanceStep()` | 保留已有离子，只添加新化合物 |
| 点击相邻下一步 | `advanceStep()` | 同上，动画不中断 |
| 点击非相邻步/往回跳 | `goToStep()` | 清空重建，进度条跳到目标时间 |

### 关键常量

```javascript
const PROGRESS = {
  x: 100,           // 距左边距
  y: CONTAINER.y + CONTAINER.h + 40,  // 容器下方
  w: 1720,          // 轨道宽度
  h: 8,             // 轨道厚度
  dotR: 10,         // 滑块圆点半径
};
```

### 步骤标签位置计算

标签按各步占总时长的比例均匀分布在轨道下方：

```javascript
let cumW = 0;
for (let i = 0; i < 3; i++) {
  const cx = x + (cumW + STEP_DURATIONS[i] / 2) / TOTAL_DURATION * w;
  // cx = 该步骤时间段的中点对应的 x 坐标
  cumW += STEP_DURATIONS[i];
}
```
